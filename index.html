4.HDFS的架构

                                           ——Hadoop Distributed File System 分布式文件系统

主从式结构： 

         主节点，只有一个 namenode

         从节点，很多个 datanodes

namenode负责：

         接收用户操作请求（命令行，java api两种方式）

         维护文件系统的目录结构 (文件夹的目录系统，分类管理)

         管理文件与block之间关系，block与datanode之间关系（block）

datanode负责：

         存储文件

         文件被分成block存储硬盘上 （丢失一部分不影响全局）

         为保证数据安全，文件会有多个副本

MapReduce的架构：

         并行计算框架 （类似项目经理和程序员的关系）

主从式结构：

         主节点，只有一个 JobTracker

         从节点，很多个 TaskTrackers

JobTracker负责：

         接收客户提交的计算任务

         把计算任务分给TaskTrackers执行

         监控TaskTrackers的执行情况

TaskTrackers负责：

         执行JobTracker分配的计算任务

5.hadoop的特点和集群特点

                                                          ——由分布式特点推出这些特点

扩容能力(Scalable)：能可靠的存储和处理千兆字节(PB)数据

成本低(Economical)：可以通过普通机器组成的服务器群来奋发以及处理数据，这些服务器群总计可达数千个节点

高效率(Efficient)：通过分发数据，hadoop可以再数据所在的节点上并行地(分布式)处理它们，这使得处理非常的快速

可靠性(Reliable)：hadoop能自动地维护数据的多份副本，并且在任务失败后能自动地重新部署计算任务
